# 🧪 브라우저 테스트 체크리스트

## ✅ 기본 기능 테스트

### 1. 메인 페이지
- [ ] `http://localhost:7777` 접속 가능
- [ ] 페이지 로딩 정상
- [ ] CSS/JS 파일 로드 확인
- [ ] 콘솔 에러 없음

### 2. 헬스 체크 엔드포인트
- [ ] `http://localhost:7777/health` 접속
- [ ] JSON 응답 확인
- [ ] 상태 값 확인:
  ```json
  {
    "timestamp": "...",
    "status": "ok",
    "checks": {
      "blog_db": { "status": "ok" },
      "mnps_db": { "status": "ok" },
      "disk_space": { "status": "ok" }
    }
  }
  ```

### 3. 블로그 기능
- [ ] `http://localhost:7777/blog` 접속
- [ ] 블로그 포스트 목록 표시
- [ ] 포스트 클릭 시 상세 페이지 이동
- [ ] 이미지 로딩 확인
- [ ] 검색 기능 작동

### 4. 에러 페이지 테스트

#### 404 에러 페이지
- [ ] 존재하지 않는 URL 접속: `http://localhost:7777/nonexistent`
- [ ] 커스텀 404 페이지 표시 확인
- [ ] "홈으로 돌아가기" 버튼 작동
- [ ] 디자인 확인 (검은 배경, 빨간 404 텍스트)

#### 500 에러 페이지
- [ ] (서버 에러 발생 시) 커스텀 500 페이지 표시 확인

### 5. API 엔드포인트 테스트

#### 블로그 API
- [ ] `GET http://localhost:7777/api/blog/posts` - 포스트 목록
- [ ] `GET http://localhost:7777/api/blog/posts/1` - 특정 포스트
- [ ] `GET http://localhost:7777/api/blog/posts?q=검색어` - 검색

### 6. 로깅 확인
- [ ] 서버 콘솔에서 요청 로그 확인
- [ ] `logs/` 폴더에 로그 파일 생성 확인
- [ ] 로그 내용 확인

### 7. 보안 설정 확인
- [ ] 개발자 도구 → Application → Cookies
- [ ] 세션 쿠키의 `HttpOnly` 플래그 확인
- [ ] (프로덕션 환경) `Secure` 플래그 확인
