# 개발 가이드라인 (Development Guide)

## 1. 프로젝트 철학 (Project Philosophy)
본 프로젝트는 **심리학(Psychology)**을 기반으로 한 **전략적 방향 전환(Strategic Pivot)** 도구입니다.
*   **목적 (Purpose)**: 단순한 성향 파악이 아니라, 사용자가 나아가야 할 **방향(Direction)**을 제시하는 것입니다.
*   **핵심 메커니즘 (Core Mechanism)**:
    *   **이상향(Ideal)**과 **잠재력(Potential)**의 간극(Gap)을 분석합니다.
    *   현재의 잠재력을 이상향으로 연결하기 위한 구체적인 **전환점(Turning Point)**을 제안합니다.

## 2. 핵심 목표 (Core Objectives)
1.  **이상향 파악 (Ideal Analysis)**: 사용자가 의식적으로 추구하는 목표 지점 설정.
2.  **잠재력 확인 (Potential Verification)**: 사용자가 무의식적으로 보유한 자원 확인.
3.  **방향 전환 제안 (Direction Strategy)**:
    *   **일치 시 (Alignment)**: 가속화(Acceleration) 전략.
    *   **불일치 시 (Gap)**: 보완 및 전환(Pivot) 전략.

## 3. 개발 원칙 (Development Rules)
*   **언어 (Language)**: 모든 문서는 **한국어**로 작성합니다.
*   **데이터 격리 (Strict Isolation)**: 외부 리소스 절대 사용 금지. 로컬 실행 보장.
*   **결과 지향 (Action-Oriented)**: 결과는 "당신은 ~입니다"가 아니라 "**~으로 전환하십시오**" 형태의 행동 강령이어야 합니다.
*   **내부 코드 사용 금지 (Prohibition of Internal Codes)**: `H-L-A`, `Type A`와 같은 시스템 내부 코드는 생성하거나 사용하지 않습니다. **(삭제 원칙)**
    *   **순수 한글 사용**: 모든 데이터 처리는 `통합적 통제 유형`과 같은 완전한 한국어 명칭으로만 이루어져야 합니다.
    *   **몰입 유지**: 피검자가 자신이 '분석 대상'이라는 느낌보다 '진단 결과' 자체에 집중하도록 기계적 기호를 원천적으로 배제합니다.
## 4. 콘텐츠 우선 (Content First)

- **핵심 원칙**: 결과 페이지의 **콘텐츠**(전략, 통찰, 액션)는 디자인보다 절대적으로 중요합니다.
- 디자인은 **보조** 역할이며, 사용자가 정보를 빠르게 이해하고 행동으로 옮길 수 있도록 지원하는 수준에 머물러야 합니다.
- 모든 UI/UX 개선은 **콘텐츠 정확성·명확성**이 확보된 이후에만 진행합니다.
- 가이드에 명시된 디자인 통일은 **시각적 일관성**을 위한 최소 요구사항이며, 콘텐츠 품질을 저해해서는 안 됩니다.
