# 브라우저 테스트 가이드

## 🚀 개발 서버 실행

개발 서버가 실행 중입니다. 다음 URL에서 접속할 수 있습니다:

**메인 URL**: http://localhost:3000

## 📋 테스트 체크리스트

### 1. 메인 페이지 (http://localhost:3000)
- [ ] 메인 페이지가 정상적으로 로드되는지 확인
- [ ] 6개의 서비스 카드가 모두 표시되는지 확인 (서비스 소개, MNPS, Second Genesis, 블로그, 게시판, 문의)
- [ ] 각 카드의 링크가 정상 작동하는지 확인
- [ ] 네비게이션 메뉴가 정상 작동하는지 확인
- [ ] 반응형 디자인 (모바일/태블릿/데스크톱) 확인

### 2. 서비스 소개 페이지 (http://localhost:3000/services)
- [ ] MNPS 서비스 소개가 표시되는지 확인
- [ ] Second Genesis 서비스 소개가 표시되는지 확인
- [ ] 각 서비스의 링크 버튼이 정상 작동하는지 확인

### 3. MNPS 페이지 (http://localhost:3000/mnps)
- [ ] MNPS 소개 페이지가 표시되는지 확인
- [ ] MNPS 테스트 시작 버튼 확인 (참고: 실제 MNPS 서버는 별도로 실행 필요)

### 4. Second Genesis (http://localhost:3000/second-genesis)
- [ ] Second Genesis 메인 페이지가 정상 로드되는지 확인
- [ ] 기존 기능이 정상 작동하는지 확인

### 5. 블로그 (http://localhost:3000/blog)
- [ ] 블로그 목록 페이지가 로드되는지 확인
- [ ] 검색 기능 테스트
- [ ] 블로그 상세 페이지 확인 (게시글이 있는 경우)
- [ ] 최신 글 사이드바 확인

**주의**: 블로그 데이터는 `apps/mnps/mnps-service/blog.db`에서 읽어옵니다. 
데이터베이스가 없거나 비어있으면 빈 목록이 표시됩니다.

### 6. 게시판 (http://localhost:3000/board)
- [ ] 게시판 목록 페이지가 로드되는지 확인
- [ ] "글쓰기" 버튼 클릭 테스트
- [ ] 게시글 작성 기능 테스트
  - 작성자, 제목, 내용 입력
  - 작성 완료 후 상세 페이지로 이동 확인
- [ ] 게시글 상세 페이지 확인
- [ ] 조회수 증가 확인
- [ ] 목록으로 돌아가기 테스트
- [ ] 페이지네이션 테스트 (게시글이 많은 경우)

**주의**: 게시판 데이터는 `apps/portal/data/board.db`에 저장됩니다.
첫 실행 시 자동으로 데이터베이스 파일이 생성됩니다.

### 7. 문의 게시판 (http://localhost:3000/contact)
- [ ] 문의 페이지가 로드되는지 확인
- [ ] 문의 작성 폼 테스트
  - 이메일 (선택사항) 입력
  - 문의 내용 입력
  - 제출 후 목록에 표시되는지 확인
- [ ] "전체 문의" / "내 문의 내역" 필터 테스트
- [ ] localStorage를 사용한 "내 문의" 추적 확인

**주의**: 문의 데이터는 `apps/portal/data/inquiries.json`에 저장됩니다.
첫 실행 시 자동으로 파일이 생성됩니다.

## 🔧 알려진 제한사항

### MNPS 서비스
- MNPS 테스트는 별도의 Express 서버(`mnps-service`)에서 실행됩니다
- `http://localhost:7777`에서 실행해야 합니다
- 현재 포털에서는 링크로 연결되어 있습니다

### 블로그
- 블로그 데이터는 `apps/mnps/mnps-service/blog.db`에서 읽어옵니다
- 데이터베이스가 없거나 경로가 잘못되면 빈 목록이 표시됩니다
- 블로그 작성 기능은 아직 구현되지 않았습니다 (기존 데이터만 조회)

### 데이터베이스 경로
- 게시판: `apps/portal/data/board.db` (자동 생성)
- 문의: `apps/portal/data/inquiries.json` (자동 생성)
- 블로그: `apps/mnps/mnps-service/blog.db` (기존 파일 사용)

## 🐛 발견된 문제 보고

테스트 중 문제를 발견하면 다음 정보를 포함해서 알려주세요:
1. 페이지 URL
2. 수행한 작업
3. 예상 동작
4. 실제 동작
5. 브라우저 콘솔 오류 (있는 경우)

## 📝 테스트 완료 후

테스트가 완료되면 다음 사항을 확인해주세요:
- 모든 링크가 정상 작동하는지
- 데이터 저장/조회가 정상 작동하는지
- 반응형 디자인이 올바르게 표시되는지
- 오류 메시지나 경고가 없는지
