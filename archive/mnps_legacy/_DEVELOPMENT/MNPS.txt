# MNPS 통합 개발 계획서 (Comprehensive Development Plan)

## 1. 프로젝트 개요 (Project Overview)

### 1.1 서비스 명칭
- **서비스명**: MNPS (Dark Tetrad Analysis Platform)
- **슬로건**: "내 안의 어둠을 과학적으로 탐구하다 (Explore Your Inner Darkness Scientifically)"

### 1.2 서비스 목적 (Service Purpose)
본 서비스의 최우선 목적은 **수익 창출**입니다.
이를 위해 우리는 **타협하지 않는 과학적 정확도(Uncompromising Scientific Accuracy)**를 추구합니다.
"대충 만든 재미용 테스트"가 아니라, **"소름 돋을 정도로 정확한 분석"**만이 사용자를 감동시키고 지갑을 열게 할 수 있습니다.
**"우리의 목적은 최대의 수익, 최고의 서비스 제공이다."**
우리는 최선을 다해 **최고의 품질(Best Quality)**을 제공하며, 그 대가로 **최대의 수익(Max Profit)**을 창출합니다.
이것이 MNPS가 시장을 지배하는 방식입니다.

### 1.3 타겟 사용자
- 자신의 잠재된 본성을 확인하고 싶어하는 **모든 잠재적 구매자**
- SNS에 자신의 독특한 결과를 공유하고 싶어하는 사용자
- 무료 결과에 만족하지 않고, 더 깊고 자극적인 분석을 위해 지갑을 열 준비가 된 사용자

### 1.4 콘텐츠 톤앤매너 (Content Tone & Manner)
- **핵심 키워드**: **도파민(Dopamine)**, **팩트 폭격(Fact-bombing)**, **유쾌한 냉소(Witty Cynicism)**
- **작성 원칙**:
    1.  **No Sugar-coating**: 위로나 공감 대신, 사용자의 본성을 적나라하게 드러내는 직설적인 화법을 사용합니다. (예: "당신은 리더십이 있군요" -> "당신은 남을 부려먹는 데 천부적인 재능이 있습니다")
    2.  **Provocative & Shareable**: SNS에 공유했을 때 친구들의 반응(놀림, 인정, 반박)을 이끌어낼 수 있는 자극적인 문구를 배치합니다.
    3.  **Authority**: 과학적 근거(논문, 통계)를 인용하되, 그 해석은 권위적이고 단정적인 어조로 신뢰감을 형성합니다.
- **목표**: 사용자가 결과에 충격을 받거나("어떻게 알았지?"), 억울해하거나("이건 아니야!"), 재미있어하며("완전 나네ㅋㅋ") 자발적으로 공유하고 결제하게 만듭니다.

### 1.5 수익화 UX 전략 (Monetization UX Strategy)
- **잠금 화면 (The Lock)**:
    - 무료 결과 페이지 하단에 **블러(Blur) 처리된 상세 분석 섹션**을 배치하여 호기심을 극대화합니다.
    - **자물쇠 아이콘**과 함께 "이 영역은 Premium 리포트에서만 확인할 수 있습니다"라는 문구를 명시합니다.
- **심리적 트리거 (FOMO)**:
    - "당신의 성향 중 가장 위험한 상위 1% 특징이 숨겨져 있습니다."와 같은 **경고성 문구**로 확인하고 싶은 욕구를 자극합니다.
    - "현재 1,240명이 이 리포트를 확인했습니다"와 같은 **사회적 증거(Social Proof)**를 노출합니다.
- **뇌과학적 몰입 전략 (Neuroscientific Engagement Strategy)**:
    - **보상 예측 오류 (Reward Prediction Error)**: 단순한 진행바 대신, 예상치 못한 타이밍에 "히든 트레이트 발견!"과 같은 시각적 보상을 터뜨려 도파민 분비를 극대화합니다. (예측 불가능한 보상이 도파민을 3배 더 자극함)
    - **인지적 종결 욕구 (Need for Cognitive Closure)**: 결과를 "98% 분석 완료" 상태에서 멈추고, 남은 2%의 결정적 정보(Critical Insight)를 가려두어 뇌가 강렬한 완결 욕구를 느끼게 합니다. (자이가르닉 효과)
    - **편도체 트리거 (Amygdala Trigger)**: "이 정보를 모르면 당신은 인간관계에서 계속 손해를 볼 수 있습니다"와 같이 손실 회피(Loss Aversion) 본능을 자극하여 결제를 '이득'이 아닌 '생존'의 문제로 프레이밍합니다.

### 1.6 킬러 콘텐츠 전략: 섀도우 마스터리 (Killer Content Strategy: Shadow Mastery)
- **핵심 가치**: 단순한 '진단'을 넘어, 자신의 어두운 본성을 사회적 성공의 도구로 활용하는 **'실전 공략집(Manual)'**을 제공합니다.
- **콘텐츠 구조 (The Premium Report)**:
    1.  **The Awakening (관점 전환)**: "사회는 당신을 '이기적'이라 부르지만, 우리는 '전략적'이라 부릅니다." (단점을 강점으로 리프레이밍)
    2.  **The Arsenal (실전 무기)**:
        - *Machiavellianism*: "정보 비대칭을 이용한 연봉 협상 필승법"
        - *Narcissism*: "근거 없는 자신감을 매력적인 퍼스널 브랜딩으로 바꾸는 기술"
        - *Psychopathy*: "감정에 휘둘리지 않고 냉철하게 손절하는 법"
    3.  **The Shield (방어 기제)**: "당신과 같은 포식자를 알아보고, 그들의 먹잇감이 되지 않는 법"
- **차별화 포인트**: 위로와 공감이 아닌, **'힘(Power)'과 '통제권(Control)'**을 팝니다.
- **자아 확증 (Identity Validation)**:
    - "이런 테스트는 결국 자기가 듣고 싶은 말을 듣는 과정입니다."
    - 우리는 그들의 욕망과 어두운 면을 '특별한 능력'으로 포장하여, 그들이 스스로를 **"남들과 다른 우월한 존재"**로 정의하도록 돕습니다. (바넘 효과의 극대화)

## 2. 개발 핵심 이론 (Core Theory)

### 2.1 어둠의 4요소 (Dark Tetrad)
본 서비스는 다음 4가지 심리적 기질을 측정하고 분석합니다.

1.  **마키아벨리즘 (Machiavellianism)**
    - 타인을 조종하고 이용하여 자신의 이익을 추구하는 성향.
    - 냉소적이고 전략적이며, 도덕적 관념보다 실리를 우선시함.
2.  **나르시시즘 (Narcissism)**
    - 자기애가 강하고 타인의 관심과 칭찬을 갈구하는 성향.
    - 우월감, 특권 의식, 공감 능력 결여가 특징.
3.  **사이코패시 (Psychopathy)**
    - 충동적이고 스릴을 추구하며, 타인의 고통에 무감각한 성향.
    - 죄책감이 없고 사회적 규범을 무시하는 경향.
4.  **사디즘 (Sadism)**
    - 타인에게 고통(신체적, 정신적)을 줌으로써 쾌감을 느끼는 성향.
## 3. 개발 시스템 및 환경 (System Architecture)

### 3.1 기술 스택 (Tech Stack)
- **Architecture**: **Error-Proof Modular MPA (Multi-Page Application)**
    - **Core Principle**: "HTML > Link > HTML" (단순함이 곧 안정성이다)
    - **Navigation**: 복잡한 JS 라우팅 대신, 표준 `<a>` 태그와 `window.location`을 사용한 물리적 페이지 이동.
    - **State Management**: `localStorage`를 활용한 영구 저장 (새로고침/오류 발생 시 데이터 보존).
- **Frontend**:
    - **HTML5**: 각 단계별 독립적인 HTML 파일 (`intro.html`, `stage1.html`, `result.html`).
    - **JavaScript**: 각 페이지에 종속된 독립 모듈 (Global Scope 오염 방지).
    - **CSS**: 페이지별 Scoped CSS + 공통 Design System.
- **Backend**:
    - **Runtime**: Node.js (Express.js) - API 및 정적 파일 서빙 전용.
    - **Database**: SQLite (데이터 무결성 보장).

### 3.2 디렉토리 구조 (Modular Design)
```
/
├── pages/                  # 독립적인 HTML 페이지 (Entry Points)
│   ├── index.html          # 메인 랜딩
│   ├── intro.html          # 검사 소개 및 동의
│   ├── stage1.html         # 검사 1단계 (마키아벨리즘)
│   ├── stage2.html         # 검사 2단계 (나르시시즘)
│   ├── stage3.html         # 검사 3단계 (사이코패시)
│   ├── stage4.html         # 검사 4단계 (사디즘)
│   └── result.html         # 결과 분석 페이지
├── modules/                # 독립적인 기능 모듈 (Import하여 사용)
│   ├── engine/             # 하이브리드 질문 엔진
│   ├── storage/            # 데이터 저장/복구 매니저
│   └── ui/                 # UI 컴포넌트 (버튼, 바)
├── assets/                 # 정적 리소스
└── server.js               # 심플한 정적 파일 서버
```

## 4. 주요 기능 명세 (Functional Requirements)

### 4.1 사용자 경험 (UX Flow)
1.  **랜딩 페이지**: 서비스 소개 및 '검사 시작' (자동 시작 파라미터 지원).
2.  **검사 진행**:
    - 몰입감 있는 전체 화면 UI.
    - 단계별 진행 (Progress Bar).
    - 중단 시 이어하기(Resume) 기능.
3.  **결과 분석**:
    - 종합 점수 및 4가지 성향별 점수 시각화 (레이더 차트).
    - 상위 % 백분위 제공.
    - 성향별 상세 해석 및 조언 제공.
4.  **회원 기능**:
    - 비회원 검사 가능 (결과 저장 시 회원가입 유도).
    - 지난 검사 이력 조회 및 변화 추이 분석.

### 4.2 관리 및 보안 (Management & Security)
- **관리자 페이지 명세 (Admin Page Specification) - "The Control Tower"**:
    - **목적**: 단순 관리가 아닌, **'수익 최적화'와 '오류 제로'**를 위한 관제탑.
    - **핵심 기능**:
        1.  **Dashboard (The Cockpit)**:
            - **Funnel Analysis**: [랜딩] -> [문항 진입] -> [잠금 화면 도달] -> [결제] 단계별 이탈률 실시간 모니터링.
            - **Revenue Pulse**: 실시간 매출, 객단가(ARPU), 결제 전환율(CVR) 시각화.
        2.  **Content Control (The Puppeteer)**:
            - **Hot-fix Editor**: 문항 오타 및 로직(가중치)을 서버 재배포 없이 즉시 수정 (JSON 기반).
            - **A/B Testing**: 잠금 화면 멘트("경고" vs "호기심")에 따른 클릭률 비교 실험 기능.
        3.  **User Insight (The Microscope)**:
            - **Individual Trace**: 특정 사용자의 모든 답변 경로 추적 (CS 대응 및 버그 분석용).
            - **Trait Map**: 전체 사용자의 성향 분포도 (마키아벨리즘 상위 10%가 30% 등) -> 콘텐츠 방향성 조정.
        4.  **System Health (The Shield)**:
            - **Error Catcher**: 클라이언트 JS 오류 실시간 수집 및 알림 (오류 제로화 필수 도구).
- **데이터 검증**: 클라이언트/서버 이중 유효성 검사.
- **오류 처리**: 전역 에러 핸들러를 통한 안정성 확보.
- **보안**: SQL Injection 방지, XSS 방지 필터 적용.
- **법적 고지 (Legal & Compliance)**:
    - **개인정보 처리방침**: 수집 항목(성향 점수, 닉네임 등) 및 보유 기간 명시.
    - **이용약관**: 서비스 면책 조항(심리 검사 결과의 의학적 효력 부인) 포함.
    - **환불 규정 (Refund Policy)**: "디지털 콘텐츠의 특성상 결제 후 열람 시 청약철회 불가" 조항 명시 (전자상거래법 제17조 2항 근거).

### 4.3 정밀 데이터 수집 (Advanced Data Collection)
- **익명 식별 시스템 (Anonymous Identification System)**:
    - **Hybrid ID**: 비로그인 사용자에게도 고유 `UUID`를 부여하고 `localStorage`와 `Cookie`에 이중 저장하여 재방문 시 식별.
    - **Digital Fingerprinting**: IP, 브라우저 정보, 해상도 등을 해시(Hash)화하여 쿠키 삭제 후에도 사용자를 추적.
- **행동 데이터 수집 (Behavioral Tracking)**:
    - **Hesitation Time**: 문항별 응답 시간(망설임)을 측정하여 '거짓 응답' 가능성 탐지.
    - **Interaction Pattern**: 마우스 호버링, 스크롤 속도 등을 분석하여 사용자의 흥미/지루함 수준 판단.
- **활용**: 식별된 익명 데이터를 기반으로, 재방문 시 "지난번엔 마키아벨리즘이 높게 나왔었죠?"와 같은 소름 돋는 개인화 메시지 제공.

### 4.4 자동화된 품질 관리 시스템 (Automated QA & Logging System: The Black Box)
- **목적**: 개발 및 운영 중 발생하는 모든 상황을 기록하고, 오류 발생 시 **'자동으로 원인을 분석하고 해결책을 제안'**하는 시스템 구축.
- **기능 명세**:

1.  **프리미엄 리포트 (Micro-transaction Model)**
    - **Price**: **3,900원** (커피 한 잔 가격).
    - **Logic**: "책 한 권"의 깊이를 기대하게 만드는 고가 정책 대신, **'가벼운 호기심'으로 결제할 수 있는 심리적 저항선(Impulse Buy Range)**을 공략합니다.
    - **Premium (유료)**: 심층 분석 리포트(PDF), 성향별 맞춤 대처법, 인간관계 조언, 상세 심리 프로파일 제공.
    - **전략**: 박리다매(High Volume, Low Price). 부담 없는 가격으로 구매 전환율(CVR)을 극대화합니다.

2.  **광고 수익 (Ad-supported Model)**
    - **방식**: 검사 결과 대기 화면이나 결과 페이지 하단에 비침해적(Non-intrusive) 배너 광고 노출.
    - **원칙**: 사용자 경험(UX)을 최우선으로 하며, 검사 몰입을 방해하지 않는 위치에 제한적으로 적용.

3.  **자발적 후원 (Donation Model)**
    - **방식**: "개발자에게 커피 한 잔 사기" (Buy Me a Coffee) 형태의 소액 후원 기능 도입.
    - **목적**: 서비스에 만족한 사용자들이 자발적으로 운영을 지원할 수 있는 창구 마련, 커뮤니티 팬덤 형성.

### 5.2 바이럴 유입 전략 (Viral Acquisition Strategy)
- **관계형 콘텐츠 (Relational Content)**:
    - **커플/친구 비교 (Compare Mode)**: "내 애인은 사이코패스일까?" 두 사람의 결과를 겹쳐서 보여주는 **'궁합(Compatibility) 분석'** 기능 제공.
    - **초대 보상 (Referral Reward)**:
        - **Logic**: 친구 3명 초대(링크 클릭 후 검사 완료) 시 'Premium 리포트' 자동 잠금 해제.
        - **Tech**: 별도 유료 솔루션 없이 **URL 파라미터(`?ref=user_id`)**와 **SQLite**만으로 자체 구현 (비용 0원).
- **공유 트리거 (Share Trigger)**:
    - **Web Share API**: 브라우저 내장 공유 기능(OS 기본 공유창)을 사용하여 개발 비용 없이 카카오톡, 문자, 인스타 공유 지원.
    - **Kakao Link SDK**: 카카오톡 공유하기 무료 API(일 30만 건)를 연동하여 썸네일이 포함된 고급 공유 카드 제공.
    - 결과 카드에 **QR코드**를 삽입하여, 오프라인 만남에서도 즉시 검사를 유도.

## 6. 상세 개발 로드맵 (Development Roadmap)

### [Phase 1] 콘텐츠 개발 및 시스템 고도화 연구 (Research & Development) - **[COMPLETED]**
- **문항 확장 (120문항)**
    - **설계 원칙 (Design Principle)**: "Simple yet Cunning" (간결하지만 교묘하게)
        - 한국인의 빠른 호흡을 고려하여 문항 길이를 최소화하고 가독성을 극대화(Instant Readability).
        - 답변은 직관적으로 유도하되, 심리적 방어 기제를 우회하도록 교묘하게 설계.
    - 4가지 척도별 30문항 개발 (기존 10문항 -> 30문항 확장)
    - 문항별 난이도, 가중치, 역채점 여부 설정
    - 심리학적 타당성 검토 및 문항 정제
- **시스템 구조 고도화**
    - 대용량 문항 데이터 처리를 위한 DB 스키마 최적화
        - `DataManager` 추상화 계층 도입 (JSON/Memory <-> SQLite 전환 용이성 확보)
    - **복합 질문 생성 (Hybrid Generation)**:
        - 가용 문항 부족 시, 5가지 패턴(Dual Category, Situational 등)을 사용하여 새로운 문항을 실시간 합성.

    - **UI**: 이탈률 최소화를 위한 **'원클릭 결제 모달(One-click Payment Modal)'** 구현.
