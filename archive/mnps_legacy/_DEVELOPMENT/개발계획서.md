# MNPS 통합 개발 계획서 (Comprehensive Development Plan)

## 1. 프로젝트 개요 (Project Overview)

### 1.1 서비스 명칭
- **서비스명**: MNPS (Dark Tetrad Analysis Platform)
- **슬로건**: "내 안의 어둠을 과학적으로 탐구하다 (Explore Your Inner Darkness Scientifically)"

### 1.2 서비스 목적 (Service Purpose)
본 서비스의 최우선 목적은 **수익 창출**입니다.
현대인의 자극적인 심리적 호기심('어둠의 4요소')을 충족시키는 고품질의 분석 도구를 제공하고, 이를 통해 **유료 결제(Premium Report)** 및 **광고 수익**을 극대화하는 것을 목표로 합니다.
교육적이거나 계몽적인 요소는 배제하며, 철저히 사용자의 흥미와 도파민을 자극하는 **엔터테인먼트형 심리 분석 서비스**를 지향합니다.

### 1.3 타겟 사용자
- 자신의 잠재된 본성을 확인하고 싶어하는 **모든 잠재적 구매자**
- SNS에 자신의 독특한 결과를 공유하고 싶어하는 사용자
- 무료 결과에 만족하지 않고, 더 깊고 자극적인 분석을 위해 지갑을 열 준비가 된 사용자

### 1.4 콘텐츠 톤앤매너 (Content Tone & Manner)
- **핵심 키워드**: **도파민(Dopamine)**, **팩트 폭격(Fact-bombing)**, **유쾌한 냉소(Witty Cynicism)**
- **작성 원칙**:
    1.  **No Sugar-coating**: 위로나 공감 대신, 사용자의 본성을 적나라하게 드러내는 직설적인 화법을 사용합니다.
    2.  **Provocative & Shareable**: SNS 공유를 유도하는 자극적인 문구 배치.
    3.  **Authority**: 과학적 근거를 인용하되, 권위적이고 단정적인 어조 사용.

## 2. 수익화 및 콘텐츠 전략 (Monetization & Content Strategy)

### 2.1 수익화 UX 전략 (Monetization UX Strategy)
- **잠금 화면 (The Lock)**:
    - 무료 결과 하단에 **블러(Blur) 처리된 상세 분석 섹션**과 **자물쇠 아이콘** 배치.
- **심리적 트리거 (FOMO)**:
    - "상위 1% 위험한 특징 발견" 등의 경고성 문구와 사회적 증거(Social Proof) 활용.
- **뇌과학적 몰입 전략**:
    - **보상 예측 오류**: 예상치 못한 타이밍의 시각적 보상.
    - **인지적 종결 욕구**: 결정적 정보를 가려 완결 욕구 자극 (자이가르닉 효과).
    - **편도체 트리거**: 손실 회피 본능 자극 ("이 정보를 모르면 손해").

### 2.2 킬러 콘텐츠: 섀도우 마스터리 (Shadow Mastery)
- **핵심 가치**: 단점을 강점으로 리프레이밍하는 '실전 공략집'.
- **The Premium Report 구성**:
    1.  **The Awakening**: 관점 전환 ("이기적" -> "전략적").
    2.  **The Arsenal**: 성향별 실전 활용법 (연봉 협상, 퍼스널 브랜딩 등).
    3.  **The Shield**: 방어 기제 및 대처법.

## 3. 시스템 아키텍처 (System Architecture)

### 3.1 기술 스택 (Tech Stack)
- **Architecture**: **Error-Proof Modular MPA** (HTML > Link > HTML)
    - 복잡한 라우팅 배제, `localStorage` 활용한 데이터 보존.
- **Frontend**: Vanilla HTML5, CSS3, JavaScript (No Framework).
- **Backend**: Node.js (Express.js) - API 및 정적 파일 서빙.
- **Database**: SQLite (`mnps.db`) - 데이터 무결성 보장.

### 3.2 데이터베이스 스키마 (`users` 테이블)
```sql
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL, -- 아이디 (이메일)
    password TEXT NOT NULL,        -- 비밀번호 (Bcrypt Hashed)
    nickname TEXT,                 -- 닉네임
    referral_code TEXT UNIQUE,     -- 추천인 코드
    referred_by TEXT,              -- 가입 시 추천해준 사람
    points INTEGER DEFAULT 0,      -- 포인트
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login DATETIME
);
```

### 3.3 API 엔드포인트
- `POST /api/auth/register`: 회원가입.
- `POST /api/auth/login`: 로그인.
- `GET /api/auth/me`: 사용자 정보 조회.
- `POST /api/results`: 검사 결과 저장.
- `GET /api/admin/stats`: 관리자 통계 (Admin Only).

## 4. 주요 기능 명세 (Functional Requirements)

### 4.1 사용자 경험 (UX Flow)
1.  **랜딩**: 서비스 소개 및 시작.
2.  **검사**: 몰입형 UI, 단계별 진행.
3.  **결과**: 레이더 차트, 요약 점수, **프리미엄 잠금**.
4.  **회원**: 이력 관리, 로그인/가입.

### 4.2 관리자 대시보드 (The Control Tower)
- **Dashboard**: Funnel 분석, 실시간 매출(Revenue Pulse).
- **Content Control**: 문항/로직 핫픽스, A/B 테스팅.
- **System Health**: 클라이언트 JS 오류 실시간 수집.

### 4.3 데이터 수집 (Data Collection)
- **행동 데이터 수집 (Behavioral Tracking)**:
    - **Hesitation Time**: 문항별 응답 시간 측정으로 거짓 응답 탐지.
    - **Interaction Pattern**: 마우스/스크롤 패턴 분석.
    *(익명 식별/핑거프린팅 기능은 제외됨)*

### 4.4 자동화된 품질 관리 (Automated QA)
- **Self-Healing**: 오류 발생 시 원인 분석 및 해결책 제안 시스템.
- **Integrity Check**: 파일 무결성 및 필수 로직 자동 검증 (`scripts/integrity_check.js`).
